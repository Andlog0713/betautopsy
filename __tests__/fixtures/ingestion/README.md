# Ingestion golden fixtures (synthetic)

Checked-in, fully synthetic CSV fixtures for the WS-TEMPORAL / WS-NUMERIC
hardening workstreams (see `ENGINE_HARDENING_OUTLINES_2026-06-10.md`).
No real user data — every row is hand-authored. Do NOT confuse with the
gitignored `test/fixtures/` (real bet data, regenerated by
`scripts/build-fixtures.ts`).

Headers use the canonical aliases from `lib/csv-parser.ts` COLUMN_ALIASES
so each file routes through `parseCSV()` → `parseRow()` unchanged.

| File | Exercises | Workstream |
|---|---|---|
| `odds-zero-and-missing.csv` | odds=0, blank odds, non-numeric odds on wins/losses — calcWhatIfProfit Infinity, impliedProb(0)=1.0 poisoning | WS-NUMERIC |
| `cash-outs.csv` | `cashed_out`/`cashed out`/`cashout` result variants carrying nonzero profit that the parser currently zeroes via the void mapping | WS-NUMERIC |
| `pushes-pending-void.csv` | pushes (win-rate denominator), pending + void with stakes (ROI denominator) | WS-NUMERIC |
| `unparseable-dates.csv` | garbage/empty/ambiguous dates that currently default to TODAY | WS-TEMPORAL |
| `cross-tz-iso.csv` | the same wall-clock moment as ISO+offset, ISO Z with millis (Pikkit-style `time_placed_iso`), and a naive string | WS-TEMPORAL |
| `naive-and-dateonly.csv` | date-only rows (midnight-UTC artifact: 65% of prod) interleaved with naive local datetimes | WS-TEMPORAL |
| `rapid-fire-unsettled.csv` | escalating stakes minutes apart on games that cannot have settled yet (placement-order "post-loss" confound), plus one genuine post-settlement chase for contrast | WS-TEMPORAL |

Expected-behavior assertions live in the outlines doc, not here; tests are
authored in the implementation PRs, not this recon branch.
